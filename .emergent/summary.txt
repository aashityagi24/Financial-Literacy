<analysis>**original_problem_statement:** The user wants to create a gamified financial literacy learning application for children (K-5) named CoinQuest. The application requires distinct user roles (Teacher, Parent, Child, Admin, School), a digital wallet with Indian Rupees (₹), a virtual store, gamified investment modules (Money Garden & Stock Market), dynamic quests, achievements, and a hierarchical content system manageable by an Admin.

**PRODUCT REQUIREMENTS:**
-   **User Roles & Core Features:** Teacher (manage class), Parent (manage chores, rewards), Child (digital wallet, store, investments), Admin (content and user management), and School.
-   **Gamified Features:** Virtual store, investment simulations, quests, achievements, and daily rewards.
-   **Content System:** A hierarchical content structure manageable by the Admin, including support for various content types (video, PDF, interactive HTML/ZIP).
-   **Teacher Repository:** A repository where admins can upload resources (images, PDFs) tagged by topic/grade for teachers to use in quests.
-   **Data Integrity & Visibility:** Content visibility must be controllable by role and grade.
-   **Word Bank/Glossary:** A section for financial literacy terms, manageable by an Admin and searchable by users.
-   **Quest Analytics:** Teachers need to see detailed responses for quests to track student performance.
-   **Unified Login:** A single sign-in/sign-up page for all user types.
-   **Content Protection:** Measures to deter casual copying or downloading of website code and uploaded content.
-   **Activity Score Tracking:** A system to capture scores from uploaded HTML activities and display them to parents and teachers.
-   **My Jobs Feature:** A new section where children can define family jobs (unpaid) and payday jobs (paid), which parents approve and manage payments for.
